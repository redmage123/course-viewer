What is Cloud Computing?

The Foundation of Modern IT

Definition

Delivery of computing services over the internet, including:

  • Servers and compute power
  • Storage and databases
  • Networking
  • Software and analytics
  • AI and machine learning

Traditional vs Cloud

Traditional Cloud
Buy hardware upfront Pay for what you use
Maintain your own servers Provider maintains
Fixed capacity Scale on demand
Weeks to provision Minutes to provision
Cloud computing = on-demand delivery of IT resources via the internet with pay-as-you-go pricing.

CapEx vs OpEx

Capital vs Operational Expenditure

üèóÔ∏è

CapEx (Capital Expenditure)

  • Upfront spending on infrastructure
  • Buy servers, data centers
  • Value reduces over time
  • Fixed costs
  • Traditional IT model
Example: Buying $1M in servers
üí≥

OpEx (Operational Expenditure)

  • Ongoing spending for services
  • Pay for what you use
  • No upfront investment
  • Variable costs
  • Cloud model
Example: $10K/month cloud bill

Cloud = Shift from CapEx to OpEx

No large upfront costs. Pay monthly based on actual usage. Scale costs with business needs.

  • Usage: Pay for what you use

Role-Based Access Control

Who Can Do What

RBAC Components

  • Security Principal: Who (user, group, service principal)
  • Role Definition: What they can do (permissions)
  • Scope: Where it applies (subscription, RG, resource)
Role Assignment = Principal + Role + Scope

Built-in Roles

Role Permissions
Owner Full access + assign roles
Contributor Full access, can't assign roles
Reader View only
User Access Admin Manage user access only
Principle of Least Privilege: Give users only the permissions they need to do their job.

Azure Policy

Enforce Organizational Standards

What Azure Policy Does

  • Define rules resources must follow
  • Evaluate existing resources
  • Prevent non-compliant resources
  • Auto-remediate violations
  • Compliance reporting

Common Policy Examples

  • Require tags on resources
  • Restrict VM sizes
  • Enforce allowed regions
  • Require encryption

Policy vs RBAC

Azure Policy RBAC
What resources can do What users can do
Resource properties User actions
Default allow Default deny
Compliance focus Access focus
Azure Policy = resource compliance. RBAC = user access. They work together but serve different purposes.

Resource Locks & Tags

Protect and Organize Resources

üîí Resource Locks

Prevent accidental changes or deletion

Lock Type Effect
CanNotDelete Can read/modify, cannot delete
ReadOnly Can read only, no changes
Locks apply to all users, even Owners!

üè∑Ô∏è Tags

Key-value pairs for organizing resources

  • Cost tracking: Department: Marketing
  • Environment: Env: Production
  • Owner: Owner: John.Doe
  • Project: Project: WebApp
Tags are NOT inherited by default. Use Policy to enforce tagging.
Resource locks override RBAC permissions. Even an Owner cannot delete a locked resource without removing the lock first.
