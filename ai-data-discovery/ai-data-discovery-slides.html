What is Data Discovery?

Data discovery is the process of finding, understanding, and cataloguing data assets across an organisation to unlock value and ensure compliance.

🔎

Identify

Locate data assets across silos, databases, file shares, cloud storage, and SaaS platforms

🏷️

Classify

Categorise data by type, sensitivity, ownership, and business relevance using AI

📚

Catalogue

Build a searchable inventory with metadata, lineage, and access controls

Key Insight: Data discovery transforms dark data into actionable intelligence. Without discovery, organisations cannot govern what they do not know exists.

ML for Data Classification

Train a classifier to automatically categorise data assets by their description.

from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.ensemble import RandomForestClassifier from sklearn.model_selection import train_test_split from sklearn.metrics import classification_report # Vectorise text descriptions tfidf = TfidfVectorizer(max_features=1000, stop_words='english') X = tfidf.fit_transform(df['description']) y = df['category'] # Train / test split X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2) # Train a RandomForest classifier clf = RandomForestClassifier(n_estimators=100, random_state=42) clf.fit(X_train, y_train) # Evaluate print(classification_report(y_test, clf.predict(X_test)))
Why TF-IDF + RandomForest? Simple, interpretable, works well on tabular metadata. For larger corpora, consider fine-tuned transformers.

Text Classification Approaches

Method Technique Accuracy Speed Best For
Regex / Rules Pattern matching Low-Medium Very Fast Known patterns (SSN, emails)
TF-IDF + ML Bag-of-words + classifier Medium-High Fast Document categorisation
Word Embeddings Word2Vec / GloVe + ML High Medium Semantic understanding
Transformers BERT / RoBERTa fine-tuned Very High Slow Complex / nuanced text
LLM Zero-Shot GPT-4 / Claude prompting High Slow Rapid prototyping, no training data
Practical Advice: Start with TF-IDF + RandomForest. Upgrade to transformers only when accuracy demands it. LLM zero-shot is great for bootstrapping labelled data.

Vector Databases for Data Discovery

Vector databases store high-dimensional embeddings, enabling semantic search — finding data by meaning, not just keywords.

from sentence_transformers import SentenceTransformer import chromadb # Create embeddings model = SentenceTransformer('all-MiniLM-L6-v2') embeddings = model.encode(descriptions) # Store in ChromaDB client = chromadb.Client() collection = client.create_collection("data_catalogue") collection.add( embeddings=embeddings.tolist(), documents=descriptions, ids=[f"asset_{i}" for i in range(len(descriptions))] ) # Semantic search results = collection.query( query_texts=["customer financial transactions"], n_results=5 )
Why Vectors? A search for “customer financial transactions” finds “payment processing records” even though no keywords match. This is the power of semantic similarity.

Vector Database Options

Database Type Scale Best For
ChromaDB Embedded / local Small-Medium Prototyping, dev, small catalogues
Pinecone Managed cloud Large Production SaaS, serverless
Weaviate Open-source / cloud Large Hybrid search, multi-modal
pgvector PostgreSQL extension Medium Existing Postgres stacks
Milvus Open-source / cloud Very Large Billion-scale vector search
Tip: Start with ChromaDB for prototyping. Migrate to Pinecone or Weaviate for production. Use pgvector if you already have PostgreSQL infrastructure.

Unsupervised Discovery with Clustering

When labels do not exist, clustering reveals natural groupings in your data assets.

from sklearn.cluster import KMeans from sklearn.decomposition import PCA import matplotlib.pyplot as plt # Cluster TF-IDF vectors kmeans = KMeans(n_clusters=5, random_state=42) clusters = kmeans.fit_predict(X_tfidf) # Reduce to 2D for visualisation pca = PCA(n_components=2) coords = pca.fit_transform(X_tfidf.toarray()) # Plot clusters plt.scatter(coords[:, 0], coords[:, 1], c=clusters, cmap='viridis') plt.title("Data Asset Clusters") plt.show()

Use Cases

  • 🔍 Discover hidden data categories
  • 📁 Group similar documents automatically
  • 🚨 Detect anomalous / misclassified assets
  • 🏷️ Bootstrap labels for supervised learning
  • 📊 Visualise the data landscape
Workflow: Cluster first, then review clusters to assign labels, then train a supervised classifier.

Detecting Sensitive Data with AI

AI can automatically scan documents and databases for sensitive information across multiple categories.

👤

PII (Personal)

  • Names & addresses
  • Social Security Numbers
  • Email addresses
  • Phone numbers
  • Date of birth
💳

Financial

  • Credit card numbers
  • Bank account details
  • Transaction records
  • Income / salary data
  • Tax identifiers
🏥

Health (PHI)

  • Medical record numbers
  • Diagnoses & conditions
  • Prescription data
  • Insurance IDs
  • Biometric data
Risk: A single undetected PII field can trigger GDPR fines of up to €20M or 4% of global turnover. Automated detection is not optional.

Hybrid Detection: Regex + ML

Combine deterministic regex patterns with probabilistic ML for maximum coverage.

Regex Patterns

import re patterns = { 'ssn': re.compile( r'\b\d{3}-\d{2}-\d{4}\b'), 'credit_card': re.compile( r'\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b'), 'email': re.compile( r'\b[\w.+-]+@[\w-]+\.[\w.]+\b'), 'phone': re.compile( r'\b\(?\d{3}\)?[\s.-]?\d{3}[\s.-]?\d{4}\b'), } def scan_text(text): findings = {} for name, pat in patterns.items(): matches = pat.findall(text) if matches: findings[name] = matches return findings

Hybrid Strategy

  • Layer 1: Regex for known formats (SSN, CC, email)
  • Layer 2: NER for names, organisations, locations
  • Layer 3: ML classifier for contextual sensitivity
  • Layer 4: LLM review for ambiguous cases
Best Practice: Regex catches the obvious. ML catches the subtle. Together they achieve 95%+ detection rates.

NER for PII Detection

Named Entity Recognition identifies people, organisations, and locations within unstructured text.

import spacy nlp = spacy.load("en_core_web_sm") text = """John Smith from Acme Corp submitted a claim on 15 March 2024. Contact: john.smith@acme.com, SSN: 123-45-6789. Based in New York.""" doc = nlp(text) # Extract named entities for ent in doc.ents: print(f"{ent.text:20} {ent.label_:10}") # Output: # John Smith PERSON # Acme Corp ORG # 15 March 2024 DATE # New York GPE
Hybrid Approach: Combine spaCy NER (for PERSON, ORG, GPE) with regex (for SSN, email) to build a comprehensive PII detector that catches both structured and contextual identifiers.

Ethics, Privacy & Bias

AI-powered data discovery introduces ethical considerations that must be addressed proactively.

🔒

Privacy

  • Discovery systems access sensitive data
  • Minimize data exposure during scanning
  • Encrypt findings and audit logs
  • Implement least-privilege access
  • Data minimisation principle
⚖️

Bias

  • ML classifiers can inherit training bias
  • Under-detection in minority languages
  • Sensitivity thresholds may vary by context
  • Regular audits of model fairness
  • Diverse validation datasets
👁️

Transparency

  • Explainable classification decisions
  • Document model limitations
  • Human-in-the-loop for high-risk
  • Clear escalation procedures
  • Stakeholder communication

Operational Challenges

Challenge Description Mitigation
Scale Petabytes of data across hundreds of systems Incremental scanning, prioritise high-risk sources
Accuracy False positives overwhelm teams; false negatives create risk Hybrid detection, human review for edge cases
Integration Diverse data sources with different APIs and formats Connector frameworks, open standards (OpenLineage)
Change Management Resistance from data owners and business units Executive sponsorship, demonstrate quick wins
Cost Compute for ML models, storage for embeddings Start small, use lightweight models, scale on ROI
Success Formula: Start with one high-value use case (e.g., GDPR PII detection), prove value, then expand scope incrementally.
