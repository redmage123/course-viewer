Microsoft Entra ID

Cloud Identity and Access Management

What is Microsoft Entra ID?

(Formerly Azure Active Directory)

  • Cloud-based identity service
  • Authentication for Microsoft 365
  • Single sign-on to thousands of apps
  • Business-to-business (B2B)
  • Business-to-customer (B2C)

Identity Types

  • User: People in your organization
  • Service Principal: Identity for applications
  • Managed Identity: Auto-managed for Azure services
  • Device: Registered or joined devices
  • External: Guests and B2B users
Entra ID vs AD DS: Entra ID is cloud-based, uses HTTP/HTTPS. AD DS is on-premises, uses LDAP/Kerberos.

Day 1

Conditional Access

IF-THEN Access Policies

Signals (Conditions)

  • User or group: Who is signing in
  • Location: Where they're signing in from
  • Device: Device state and compliance
  • Application: Which app they're accessing
  • Risk: Sign-in or user risk level
  • Device platform: iOS, Android, Windows

Decisions (Actions)

  • Block access: Deny completely
  • Grant access: With conditions:
    • Require MFA
    • Require compliant device
    • Require hybrid joined device
    • Require approved app
  • Session controls: Limit session
Conditional Access = IF (conditions) THEN (access decision). It's the Zero Trust policy engine.

Microsoft Entra ID Protection

Risk-Based Identity Protection

User Risk

Probability that an identity is compromised

  • Leaked credentials
  • Threat intelligence
  • Unusual behavior patterns

Sign-in Risk

Probability that a sign-in is unauthorized

  • Anonymous IP address
  • Atypical travel
  • Malware-linked IP
  • Unfamiliar sign-in properties

Risk-Based Policies

  • Low risk: Monitor
  • Medium risk: Require MFA
  • High risk: Require password change or block
Self-Remediation: Users can reset their own passwords or complete MFA to clear risk.

Practice Questions

Test Your Knowledge

Q1: Which Zero Trust principle states users should have minimum necessary permissions?

A) Verify explicitly   B) Assume breach   C) Least privilege access   D) Defense in depth

Answer: C - Least privilege access

Q2: Which service provides cloud-native SIEM and SOAR capabilities?

A) Defender for Cloud   B) Microsoft Sentinel   C) Defender XDR   D) Entra ID Protection

Answer: B - Microsoft Sentinel

Q3: What does Conditional Access use to make access decisions?

A) User location only   B) Password strength   C) Signals/conditions   D) Static rules

Answer: C - Signals/conditions (user, location, device, app, risk)

Q4: Which portal provides Microsoft's compliance audit reports?

A) Compliance Manager   B) Service Trust Portal   C) Defender Portal   D) Purview Portal

Answer: B - Service Trust Portal (servicetrust.microsoft.com)
