Functions: Reusable Code

Functions let you write code once and use it many times:

Defining Functions
# Simple function def greet(name): return f"Hello, {name}!" print(greet("Alice")) # Hello, Alice! # Default parameters def greet(name, greeting="Hello"): return f"{greeting}, {name}!" print(greet("Bob")) # Hello, Bob! print(greet("Bob", "Hi")) # Hi, Bob! # Multiple return values def get_stats(numbers): return min(numbers), max(numbers) low, high = get_stats([1, 5, 3, 9, 2])

Why Use Functions?

  • DRY: Don't Repeat Yourself
  • Organize: Break code into logical pieces
  • Reuse: Write once, use many times
  • Test: Easier to test small functions

Good function names describe what the function does: calculate_total(), send_email(), get_user()

Sets: Unique Collections

Sets store unique values with no duplicates and no order:

Creating and Modifying Sets
# Creating sets fruits = {"apple", "banana", "cherry"} numbers = {1, 2, 3, 4, 5} # Duplicates are automatically removed duplicates = {1, 2, 2, 3, 3, 3} print(duplicates) # {1, 2, 3} # Adding and removing fruits.add("orange") fruits.add("apple") # No effect (exists) fruits.remove("banana") fruits.discard("grape") # Safe (no error) # Membership testing (very fast!) if "apple" in fruits: print("Found it!")

Set Operations

a = {1, 2, 3, 4} b = {3, 4, 5, 6} # Union (combine) print(a | b) # {1, 2, 3, 4, 5, 6} # Intersection (common) print(a & b) # {3, 4} # Difference (in a, not in b) print(a - b) # {1, 2} # Symmetric difference print(a ^ b) # {1, 2, 5, 6}
Note: Sets are unordered - you can't use indexing like fruits[0]!

None: The Absence of Value

None is Python's way of representing "nothing" or "no value":

Using None
# None as default parameter def greet(name=None): if name is None: return "Hello, stranger!" return f"Hello, {name}!" print(greet()) # Hello, stranger! print(greet("Alice")) # Hello, Alice! # Functions return None by default def do_something(): print("Doing...") # No return statement result = do_something() print(result) # None # Checking for None (use 'is', not '==') value = None if value is None: # Correct print("No value")

Common Use Cases

  • Optional parameters: Default function arguments
  • Initialization: Variables not yet assigned
  • Return values: No meaningful result
  • Database nulls: Missing or unknown data

Best Practice: Always use is None instead of == None

None is falsy, so if not value: works but also catches 0, empty strings, and empty lists!

Variable Scope: Where Variables Live

LEGB Rule

Python looks for variables in this order:

  • Local: Inside current function
  • Enclosing: In parent functions
  • Global: Module level
  • Built-in: Python's built-ins
x = "global" # Global scope def outer(): x = "enclosing" # Enclosing def inner(): x = "local" # Local print(x) # "local" inner() print(x) # "enclosing" outer() print(x) # "global"

Local vs Global

# Local variable (preferred!) def calc_total(price, tax): total = price * (1 + tax) return total # Local only # Global variable (avoid!) count = 0 def increment(): global count # Required! count += 1
Avoid global variables! They make code hard to test and debug. Pass values as parameters instead.
Best practice: Keep variables local. Use function parameters and return values.
